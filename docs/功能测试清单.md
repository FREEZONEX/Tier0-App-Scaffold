# HMI 工艺画面（SCADA Mimic）功能测试清单

> 面向测试人员的功能点 + 注意点参考文档。
> 每个功能点包含：**操作 / 预期 / 注意点（易错与边界）**。
> 末尾附 `data-testid` 速查、关键常量边界值、已知未实现项。

---

## 0. 一句话概览（先读这段）

本应用是一个 **工业工艺流程画面（HMI Mimic）组态 + 监控** 单页应用：

- 核心页面是 `/`，一块**画布**，可拖元件、连线、绑 MQTT 信号，做成工艺流程图。
- 画面有 **三种模式**：`编辑` / `预览（只读监控）` / `演示（钉死只读样板）`。模式决定 90% 功能是否出现。
- 数据来自 **MQTT 实时信号**（通过平台 SDK over WebSocket）；没配 broker 时退化为 **mock 仿真**；空库首屏退化为 **演示样板**。
- 画面存进 **Postgres**（自动保存 + 手动保存），目前 UI 只暴露**单张默认图**。
- 鉴权靠**平台网关注入身份**，本应用不管密码。只有 **admin 能编辑**，其余角色全只读。

**测试前必须先确认当前处于哪个模式、哪个数据源**（右上角徽标），否则会把"功能没出现"误判为 bug。

## 1. 三态模式：编辑 / 预览 / 演示（最先搞清，决定其余一切）

| 模式 | 何时进入 | 可编辑 | 数据源 | 落库 |
|---|---|:---:|---|:---:|
| 编辑 edit | admin 默认（DB 有真实图时） | ✅ | real / mock | ✅ 自动+手动 |
| 预览 preview | operator 固定；admin 可切 | ❌ 只读监控 | real / mock | ❌ |
| 演示 demo | DB 真实图为空时首屏默认 | ❌ 钉死只读样板 | mock + 演示徽标 | ❌ 绝不落库 |

**操作**：admin 顶栏点「编辑 / 预览 / 演示」切换（`mode-toggle` / `mode-demo`）。

**预期**：
- 只有**编辑态**才出现：元件库 Palette、底部编辑工具条、多选条、Inspector 可写、节点可拖/缩放/连线。
- 预览/演示态：工具强制为「平移」，节点不可动，Inspector 只读。

**注意点（高优先级）**：
1. **「演示」按钮仅当 DB 真实图为空时出现**。在编辑态画出第一个节点并落库后，**演示按钮立即消失、无需刷新**。→ 必测「演示切编辑后画一个元件，演示标签应消失」。
2. **演示态绝不落库**，任何操作都不应改 DB 行。
3. **非 admin 角色**（operator 等）：首屏强制预览，无模式切换按钮、无工具条、无元件库，全只读。
4. `VITE_DEMO_EDIT=1` 是本地特例（演示图变可编辑、走「保存画布」写回 JSON 文件）。**生产 build 不带此 env，演示恒只读**——写源文件会被服务端二次门控拒绝。
5. 环境/数据源判定会**明文打到浏览器 Console**（`[tier0-config]` / `[MQTT] 建连配置`），可据此确认当前是 real / mock / demo。

## 2. 画布交互：平移 / 缩放 / 适应视图

### 2.1 平移 Pan
- **操作**：① 平移工具拖空白；② 任意工具中键拖；③ 任意工具**按住空格 + 左键拖**。
- **预期**：画布跟手平移，光标 grab / grabbing。
- **注意点**：在 input/textarea（如改元件名）里打空格**不触发**平移（已规避）；空格平移松开后状态应正确复位。

### 2.2 缩放 Zoom
- **操作**：滚轮（以光标为锚）；右下 +/− 按钮（以画布中心为锚）。
- **预期**：缩放钳制 **0.05 ~ 12 倍**；滚轮在画布上**不滚动页面**（离开画布恢复）。
- **注意点**：缩放到极限继续滚不应抖动/翻转。

### 2.3 适应视图 Fit
- **操作**：双击画布空白或图元、右下「适应视图」按钮。
- **预期**：按场景包围盒居中；**最大放大倍率=1**（稀疏图不会被放大到超过设计尺寸）；底部预留 38px 给位号文字不被裁。
- **注意点**：① **双击设备的动作按钮区不触发 Fit**（给预览连点动作让路）；② 手动平移/缩放过后，容器 resize 不再自动重新适应；③ 从报警条点条目会把该设备居中。

## 3. 选择：单选 / 框选 / 多选

| 方式 | 操作 | 预期 | 注意点 |
|---|---|---|---|
| 单选 | 点图元 | 选中并打开右侧 Inspector；点空白清空 | 拖拽末端的点击被抑制（拖完不会误选/误清）；节点选择与连线选择互斥 |
| 框选 | 选择工具拖空白 / **任意工具 Shift+拖** | 按选框 bbox 收集节点 + 与折线相交收集连线 | **框选按 bbox 近似，忽略旋转/圆形精修**——旋转/圆形元件可能"擦边"被选中（已知近似）；移动 <3px 不算拖拽 |
| 多选 | 框选 ≥2 个 | 顶部浮出「已选 N 个」条 + 删除/取消 | 单选不显此条（单选删除走 Inspector） |

## 4. 键盘快捷键（两套监听，作用域不同——重点易错区）

| 快捷键 | 作用 | 作用域 |
|---|---|---|
| `Cmd/Ctrl+Z` | 撤销 | **全局**（任意焦点可用） |
| `Cmd/Ctrl+Shift+Z` 或 `Cmd/Ctrl+Y` | 重做 | **全局** |
| `Esc` | 取消进行中拖拽/连线 → 否则清空选择 | 画布聚焦 |
| `Delete` / `Backspace` | 优先删选中连线，否则删选中节点 | 画布聚焦 |
| `Cmd/Ctrl+A` | 全选所有节点 | **画布聚焦** |
| `Cmd/Ctrl+D` | 复制选中（偏移 30px、自动新位号、**不带连线**） | **画布聚焦** |
| `Cmd/Ctrl+C` / `Cmd/Ctrl+V` | 内存剪贴板复制 / 粘贴（+30px） | **画布聚焦** |
| 方向键 / Home / End | 节点间循环移动单选游标 | **画布聚焦** |

**注意点（高优先级回归）**：
1. **作用域陷阱**：`Cmd+Z`/重做随处可用；但 `Cmd+A / Cmd+D / Cmd+C/V / Del / 方向键` **必须画布处于焦点**才生效。先点了 Inspector/工具条再按这些键会无效——是预期但极易误报。
2. 剪贴板是**内存级、仅当前标签页**，刷新即丢。
3. Mac/Win 双修饰键都支持。
4. 在 input/textarea 内不拦截撤销/重做（留给原生）。

## 5. 元件放置 / 拖拽移动 / 智能对齐

### 5.1 放置
- **操作**：左上「元件」展开元件库 → ① 拖元件卡到落点；② 点元件卡 → 在**视口中心**新建。
- **预期**：新建后自动选中、打开 Inspector 直接配置；id 形如 `pump-1`、`pump-2`（按类型递增、跳过已占用）。
- **注意点**：① 元件库只在编辑态出现；② 点击放置落"当前视口中心"，缩放/平移后中心点会变；③ 悬停元件项弹详情卡，底部元件的卡片会上移避免被裁——小屏/底部元件要验证不被裁。

### 5.2 拖拽移动
- **操作**：编辑态按住图元拖动。命中节点在选择集内（≥2）→ 整组移动；否则只动它。
- **预期**：连线端点跟随移动的节点端。
- **注意点**：**编辑态下平移工具也能拖动元件**（命中节点即移动，不分 pan/select）。pan 与 select 的真正区别只在「拖空白」（pan=平移画布 / select=框选）。

### 5.3 智能对齐（Figma 式，**无网格**）
- **操作**：拖**单个**节点时，与其他节点中心 x/y 接近（≤10px）则吸附到中线，并画贯穿画布的虚线参考线。
- **注意点**：① 只对单节点生效，**多选整组拖不吸附、不显参考线**；② 松手参考线收起；③ **本编辑器没有网格、也没有网格吸附**——不要期待网格点阵。

---

## 6. 连线 / 边（端口钉死 + 自由虚线端点）

### 6.1 从端口拉线连边
- **操作**：编辑态光标移到图元上 → 显示四向连接点（最近的放大描环）→ 从连接点（9px 内）按下拖出橡皮筋 → 落另一图元=连边；落空白=node→自由点边；落回自身=取消。
- **预期**：
  - **起止端口钉死**：连线记住起手方位和落点方位，**端口位置绝不随对端移动而变**。
  - **实/虚线自动判定**：任一端是仪表 instrument → **虚线引线**；否则**实线管道**。**无手动切换**。
  - 泵/阀只显示/可连 **左右口**，不含上下。
- **注意点**：拉线在 pan 和 select 工具都可用（只要编辑态），不需要单独"连线模式"。"画时所见=提交所得"。

### 6.2 自由虚线/管道（独立线段）
- **操作**：元件库「连线」小节拖入/点选 → 落点生成**长 120px、两端均为自由点**的水平线段。
- **注意点**：这是**唯一两端可任意拖动的边**，端点不钉在任何图元上。

### 6.3 拖动连线端点改端
- **操作**：选中单条边 → 两端出现实心圆手柄 → 拖端点。落图元=绑定到最近边；落空白=自由点；**按住 Alt = 即便压在元件上也强制自由放**。
- **注意点**：端点**绝不吸附到其他管道/线**（故意，避免重合看不清流向）；吸附目标在松手时才锁定；拖拽中按 Esc 可复原。

### 6.4 选中 / 删除连线
- 编辑态点击连线选中（容差 6px），高亮 + Del 删除；框选可多选连线。删除节点会**级联删除连接它的连线**，可撤销。

## 7. 缩放手柄（等比拉伸单个元件）

- **操作**：编辑态 + 选择工具 + **恰好选中 1 个节点**时，出现虚线框 + 四角抓点，拖角缩放。
- **预期**：**等比缩放（锁纵横比）**，绕中心；倍率钳制 **0.25 ~ 5**。
- **注意点**：① 多选或平移工具下不显手柄；② 拉伸整段合并为一步历史；③ Inspector 也有尺寸数值输入，两条路径结果要一致；④ 拖拽中 Esc 复原。

---

## 8. 撤销 / 重做 / 取消拖拽 / 删除

- **一次拖拽 = 一步历史**：连续拖动只产生一个可撤销步（拖拽中每帧 replace 不入栈）。
- **commit 幂等短路**：新值=旧值时跳过、不污染撤销栈（如失焦未改）。新 commit 清空 redo。
- **Esc 取消进行中拖拽**：拖节点/缩放/拖端点过程中 Esc → 弹回拖拽前快照；优先级高于"清选择"。
- **注意点**：① 撤销/重做按钮空栈时禁用；② **历史上限 50 步**，超出丢最旧——长时间连续编辑后无法撤销到最初；③ 触摸/触控板手势（pointercancel）会清掉拖拽残留态，触摸设备要专门回归；④ 删除可撤销，撤销后连线也应恢复。

---

## 9. 设备动作按钮（编辑态也会真实下发——⚠️ 重点警示）

- **操作**：图元下方的动作按钮，按下→在同一按钮内松手 → 触发；溢出 `⋯` 开菜单。
- **⚠️ 注意点（CAUTION）**：
  1. **编辑和预览两态点击动作按钮都会执行动作（向 broker 发布 MQTT 消息）**，不是只在预览态。**编辑模式下误点设备动作按钮区可能真实下发指令**。
  2. 带 `confirm` 的动作会弹二次确认框。
  3. **执行动作无本地回显**——图元只反映 broker 回来的真实数据（不会"发了就假装更新"）。
  4. 有「已下发」（sent，1.5s 自动清）/「按住」（pressed）视觉反馈。
  5. 配置动作的入口在 Inspector「配置操作…」，不在画布。

## 10. 元件库内容（28 种元件 + 2 种连线）

按品类（Palette 固定顺序：设备 / 执行器 / 容器 / 换热 / 仪表 / 端子）：

| 品类 | 类型 | 可绑状态字段 |
|---|---|---|
| 设备 | pump 泵、motor 电机、fan 风机、compressor 压缩机、agitator 搅拌器 | running + rpm |

| 设备 | **mixer 静态混合器** | **无状态字段**（特殊） |
| 执行器 | valve 阀、damper 风门、checkvalve 止回阀、safetyvalve 安全阀 | open |
| 执行器 | switch 开关、controlvalve 调节阀 | opening 0–100 |
| 容器 | tank 罐、vessel 立式容器、drum 卧式罐、silo 料仓、cyclone 旋风分离器 | level |
| 容器 | column 精馏塔 | level + temp |
| 换热 | exchanger 换热器、condenser 冷凝器 | temp |
| 换热 | cooler 空冷器、heater 加热器 | running + temp |
| 仪表 | meter 流量计 | flow |
| 仪表 | dialgauge 表盘、bargauge 条形、instrument ISA 仪表、readout 数值点 | value |

- 未注册的 type 渲染为虚线方框 + "?" 占位（不崩图），每个未知 type 只告警一次。

## 11. 元件属性面板 Inspector

### 11.1 名称
- 改名输入框，回车/失焦保存；空白回落显示 `node.id`。只读态名称变纯文本。

### 11.2 变换（大小 / 旋转）
- 大小滑块 **0.25–5、步进 0.05**；旋转步进 15°、+90° 按钮，规整到 `[0,360)`。
- **注意点**：UI 只暴露**等比缩放**（sizeX=sizeY）；schema 支持分轴拉伸但 UI 不暴露。

### 11.3 静态属性（仅部分类型）
- 位号 tag（instrument，placeholder `FT/LT/PT…`）；样式 display（方框 ⇄ 圆气泡，互斥）。
- **注意点（潜在盲区）**：Inspector 目前只渲染 `tag` 和 `display` 两个控件。`vessel.agitator`、`dialgauge.face` 等 schema 支持的属性**在面板里没有编辑控件**，只能靠上传 JSON 设置。

### 11.4 实时数据 / 趋势 / 历史
- 实时数据区列出所有绑定字段值 + 额外数据点，**越限变色**（alarm 红加粗 / warn 黄加粗）。
- 主数值显示 Sparkline；百分比类字段（level/value/opening）额外显示环表。
- 「查看历史数据」按钮**编辑/只读态都可用**（纯查询）。

### 11.5 删除元件
- 底部红色「删除元件」→ 删节点 + 级联删引用它的连线。

## 12. 设备绑定 / 信号映射（产品核心：每工厂 MQTT 标准不可知）

> 核心理念：**不标准化设备，标准化"映射机制"**。三层严格分离：
> **数据绑定**（数据→外观）、**操作**（写值动作）、**额外数据点**（纯显示）。

### 12.1 订阅 Topic（TopicManager）
- **操作**：「订阅 Topic」输入框边输边搜 UNS（防抖 150ms），下拉点选或回车手填。
- **注意点（重要边界）**：
  1. **删除 topic 会一并清除引用它的字段绑定**，但保留其他 topic 的绑定。→ 必测。
  2. 配字段绑定时**自动把 topic 加入订阅列表**——不手动加 topic 也能收到数据。
  3. 无平台 env（`available:false`）时下拉不弹，退化为纯手填。

### 12.2 字段来源（FieldSource：先选 topic 再选字段）
- 字段下拉来源：实时报文扁平路径 > UNS schema 记忆字段；都没有 → 退化为**手输框**（仍可配）。
- **注意点**：① 换 topic 自动清空字段；② "选了 topic 还没选字段"是合法中间态，**此时绑定尚未落库**——切走再回来不应残留半个绑定；③ path 支持 `a.b[0].c`，任一环缺失返回 undefined 不抛异常。

### 12.3 布尔字段映射（StateValueMap）——成对独立可编辑（用户强诉求，必测）
- **操作**：两个状态图标（如阀"开/关"），右边各填条件 = 运算符（= ≠ > < ≥ ≤）+ 值。
- **预期语义（关键）**：
  - `test`（开条件）/ `testOff`（关条件）**各自独立、互不锁定**。
  - 命中 test→真；命中 testOff→假；**只配其一→其余取补**；**两者都配且都不命中→未知（quality=unknown，不静默猜）**。
  - eq/ne 支持逗号多值（中英文逗号都分割）；> < ≥ ≤ 走数值比较。
- **注意点**：`invert`（布尔取反）schema 支持但 **UI 未暴露**；连续快速输入不应失焦。

### 12.4 数值字段映射（量程 + 单位 + 告警）
- **量程 scale**（min/max，默认 0/100）：**仅驱动视觉填充比例/仪表角度，显示仍是真实值，不参与判阈**。
- **单位 unit**：手填，纯显示，不内置推断。
- **告警阈值 HIHI/HI/LO/LOLO**：
  - **判原始值**（非 scale 归一后）；`value ≥ HIHI 或 ≤ LOLO → alarm`；`value ≥ HI 或 ≤ LO → warn`。
  - 边界是 `>=` / `<=`（含等于），**必测临界值**。
  - 四格全空 = 清除阈值。

### 12.5 额外数据点（WatchManager，纯显示）
- **操作**：「添加数据点」→ 填名称 + 选 topic/字段 + 可选阈值。topic 选齐才落库。
- **⚠️ 已知坑（必测）**：**空字段路径会取整条 payload → 数值转换 NaN → 阈值永不触发**。界面会显黄字警告（`watch-alarm-nopath-warn`）。测：配了 watch 阈值但没选具体字段，验证警告出现且阈值确实不触发。

### 12.6 操作配置（ActionsEditor，写值动作）
- **操作**：Inspector「配置操作…」弹窗 → 加操作（按钮文字 + topic+payload 消息 + 可选发送前确认）。
- **注意点**：① 单设备动作上限 **8**；② 列表顺序=优先级，≤3 个全图上直达、≥4 个前 2 直达其余进 `⋯` 菜单；③ 选 topic 后自动回填示例 payload（仅当 payload 仍是默认空值）；④ payload 非合法 JSON 提示「将按原文发送」；⑤「试发送」按钮（admin 调试）不弹确认直发。

## 13. 元件渲染状态（运行 / 报警 / 失联 / 未配置）

| 状态 | 视觉 | 触发条件 |
|---|---|---|
| 活跃（运行/开/通） | 深填充·醒目 | 布尔=真 |
| 非活跃（停/关/断） | 浅填充 | 布尔=假 |
| 故障 fault | **红环闪烁 + ! 角标** | 值 ≥ HIHI 或 ≤ LOLO |
| 预警 warn | 琥珀环·不闪 | 值 ≥ HI 或 ≤ LO |
| 未配置 | **虚化·实线·无角标** | 有可绑字段却一个没绑、且无动作 |
| 失联 stale | **虚化·虚线 + ? 角标** | 有订阅 topic 但取不到任何值 / 未连 MQTT |
| 选中 | 青色选中环 | 被选中 |
| 联锁 | 挂锁角标 | 绑定名为 `interlock` 的字段为真（见 §16） |

**装饰优先级**：未配置 > 不可信(失联/未知) > 故障 > 预警 > 选中。

**注意点**：
1. **故障不再读单独的 fault 布尔位**，纯由数值越 HIHI/LOLO 派生。→ 要触发红环必须配 HIHI 或 LOLO 阈值并越限。
2. 配了动作按钮即算"已配置"，不再虚化；mixer 永不判未配置。
3. **terminal 端子永不褪色虚化**（去掉了未配置/失联态），始终实色实线。
4. 失联时液面波动/自转等动画**冻结**。
5. **管线流动动画当前全局关闭**——连线不要期待流动效果。故障闪烁、设备自转、储罐液面波动仍在。

## 14. 实时数据 / MQTT 连接

### 14.1 连接来源
- broker 主机取自 **`/openapi/v1/info` 接口的 `data.mqttBroker`**；端口回退 env `TIER0_MQTT_PORT` 或默认 **8084**；SDK over WebSocket(wss) 直连。
- 配置只走服务端无前缀 env `TIER0_*`，客户端经 server function 取回。

### 14.2 连接状态徽标
- 状态机：disconnected / connecting / connected / error。徽标：绿=已连、红=断开/错误、琥珀脉冲=连接中；demo/mock 走另一分支（蓝/琥珀静态点）。

### 14.3 订阅 / retain（核心回归点）
- **差量订阅**：只对真正新增的 topic 订阅（触发 retain 补推当前值），移除的退订，已订阅的原封不动。
- **连接建立后保持，不随 schema 重建**。
- **⚠️ 必测高风险回归点**：**改名/移动/调阈值/拖拽等不动 topic 的编辑，必须零订阅变化、零重连**。若出现"全图读数瞬间跳回旧值"= 误触发了重连/重订阅（bug）。
- 切出演示模式（画第一个真节点）→ mock 源销毁、重建真实源，**这是有意重连**，区别于编辑误重连。

### 14.4 边界
- **无数据 = 失联（stale）**：订阅了但 broker 没数据 → 图元虚化，而非空白。
- payload 先尝试 JSON.parse，失败按原始字符串（非 JSON 标量也能用）。
- 静态/低频 topic 靠 **UNS 首帧回填**（连接时拉一次当前值），UNS 不可用则纯靠 MQTT 可能一直失联。
- apiKey 格式不符 → 凭证解析失败 → 连上又被踢（看 Console `凭证解析:失败`）。
- broker 取不到时**静默回退** env 旧值（看 Console `info fetch failed`），不报错。

## 15. 报警条 AlarmStrip

- **触发**：唯一来源 = **阈值越限**（无独立报警位），基于原始工程值。节点 fault = 最严等级达 alarm。
- **显示**：常态绿「无报警」胶囊；有越限变红/琥珀计数胶囊（`N 报警 · M 预警`），有报警时整条脉冲。点开看全表，点某条把该设备选中并居中。
- **排序**：alarm 在 warn 之前，再按设备名、字段名稳定排序。

**⚠️ 已知缺口（务必覆盖 / 如实反馈）**：
1. **没有"确认（Acknowledge）"功能**——无 ack / 消音 / 已确认态。若需求文档写了"确认"，这是**未实现项**。
2. **没有去抖 / 死区 / 迟滞**——阈值是**瞬时判定**，值在阈值边缘抖动会导致报警**反复进出/闪烁**（当前设计，无抑制）。
3. **配了 map 但原始值未命中** → quality=unknown，**不计入报警**——别把 unknown 当报警。

## 17. 联锁引擎 Interlock —— ⚠️ 运行时未接线（务必如实反馈）

- `interlocks` 规则当前**只做组态期校验**（环检测 + 引用完整性，产出非阻断警告横幅），**运行时没有任何代码评估规则去自动锁节点**。
- **配置一条联锁规则（条件满足）→ 目标节点不会被自动锁、不出挂锁角标。** 若需求要求联锁联动闭锁/报警，这是**核心未实现项**。
- 当前**唯一能点亮挂锁角标的方式** = 给图元**绑定一个名为 `interlock` 的字段**且其 MQTT 值为真；且角标无 trip/inhibit 区分（差异变体走不到）。
- 组态校验仍生效：联锁引用不存在的节点 → 黄色警告；链式规则成环 → 警告列出环路径。

## 18. 画面持久化（保存 / 加载 / 改名）

- **数据模型**：Postgres `mimics` 表，`data` 列存整张 JSON。注意有**两个名字**：DB 列 `name` 与 JSON 内 `data.meta.name`。
- **自动保存**：编辑态任何变更**防抖 800ms** 落库；跳过首屏加载；演示态绝不自动落库；**失败静默吞掉**（无 UI 提示）。
- **手动保存**「保存画布」按钮：有 idle/saving/saved(2s)/error(3s) 四态文案。
- **⚠️ 改名坑（必测）**：UI 改名只改 `data.meta.name`，**DB `name` 列仍是 "default" 不变**。后果：① 刷新后标题保留（来自 JSON）；② 默认图查找不受影响。后端有独立 `renameMimicFn` 写 `name` 列但 **UI 从不调用**。
- **单图 vs 多图**：后端 CRUD 齐全，但 **UI 只暴露单张 default 图**（多图切换/新建/删除隐藏）。
- **空库首屏**：seed 一条空白 default 图 → nodes 为空 → 进**演示模式**（即使 admin）。
- **客户图生成只走脚本** `tsx scripts/load-mimic.mts <schema.json>`（UPSERT 进 `name='default'` 行），是"AI 生成图 → DB"的唯一通道。
- **校验上限**：nodes ≤ 2000、edges ≤ 5000、interlocks ≤ 2000；节点 id 必须唯一；校验失败返回字段级错误、**绝不返回半张图**。

## 20. 登录鉴权（SSO 网关）

鉴权链（按此顺序构造用例）：
1. **CSRF**：POST/PUT/PATCH/DELETE 非同源 → **403**。
2. **公开路径**直接放行：`/login`、`/api/auth`、`/api/health`、`/api/manifest`、`/_serverFn` 等。
3. 已有 `mes-session` cookie → 放行。
4. 无 cookie + 网关给了**有效 role** → 中间件直接签 cookie 并 302 回原 URL，**用户永远看不到 /login**。
5. 无 cookie + 有 user 但 role 缺失/无效 → 302 到 `/login?from=…`。
6. 无 cookie + 无网关头 → **401 "Platform authentication required"**。

**注意点**：
- 网关身份支持 3 种 header 格式，最少需要一个 user ID。
- **只有 admin（含 `edit_mimic` 权限）能编辑**，未知角色一律静默拒绝。
- `SESSION_SECRET`：生产必须 ≥32 字符否则启动失败；**dev 未设则每次重启登录态失效**（正常）。
- **⚠️ 安全测试关注**：mimic 的 server functions（save/load/create/rename/delete）**不做角色级鉴权**，`/_serverFn` 在公开白名单里，只受同源 CSRF 限制（POST 类）。需覆盖带/不带 cookie、跨源调用的差异。
- `/api/auth/logout` 存在但**无 UI 入口**，只能直接调 API。

## 25. 已知未实现 / 易误判项（汇总，测试报告前必读）

1. **联锁运行时未接线**：`interlocks` 规则不产生运行时闭锁，唯一生效路径是绑名为 `interlock` 的字段。
2. **报警无确认（Acknowledge）功能**，无去抖/死区——边缘抖动会反复报警。
3. **历史趋势已移除归一化**：多量纲叠加共享原始值 Y 轴，小量纲线显平=预期。
4. **管线流动动画全局关闭**。
5. **改名只改 JSON 名不改 DB `name` 列**。
6. **多图/新建/删除/语言切换/登出 UI 均隐藏**（后端能力保留）。
7. **/login 不是角色选择界面**，直接发 admin session。
8. **mimic server fn 不做鉴权**（安全关注点）。
9. **英文模式出现中文 = 漏翻降级**，非 bug。
10. **/station、/review、/monitor 是空壳**，主区空白属正常。
11. **Inspector 只暴露 tag/display 两个静态属性控件**，其余属性需上传 JSON。
12. **空字段路径的 watch 阈值永不触发**（有黄字警告）。
13. **编辑态点动作按钮也会真实下发 MQTT 指令**。
